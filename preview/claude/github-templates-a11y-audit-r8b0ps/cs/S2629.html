S2629 — Logging templates should be constant

Branch preview for claude/github-templates-a11y-audit-r8b0ps — not the production site.

Language
C#
Type
Code smell
Severity
Major
Tags
performance, logging

Why is this an issue?

Logging arguments should not require evaluation in order to avoid unnecessary performance overhead. When passing concatenated strings or string interpolations directly into a logging method, the evaluation of these expressions occurs every time the logging method is called, regardless of the log level. This can lead to inefficient code execution and increased resource consumption.

Instead, it is recommended to use the overload of the logger that accepts a log format and its arguments as separate parameters. By separating the log format from the arguments, the evaluation of expressions can be deferred until it is necessary, based on the log level. This approach improves performance by reducing unnecessary evaluations and ensures that logging statements are only evaluated when needed.

Furthermore, using a constant log format enhances observability and facilitates searchability in log aggregation and monitoring software.

The rule covers the following logging frameworks:

  • Microsoft.Extensions.Logging
  • Castle.Core
  • log4net
  • Serilog
  • Nlog

How to fix it

Use an overload that takes the log format and the parameters as separate arguments. The log format should be a constant string.

For log4net and Castle.Core, use a format method (e.g. DebugFormat, InfoFormat) with indexed placeholders ({0}, {1}, …​).

Code examples

Noncompliant code example


msLogger.LogDebug($"The value of the parameter is: {parameter}.");      // Noncompliant - Microsoft.Extensions.Logging
log4netLogger.Debug($"The value of the parameter is: {parameter}.");    // Noncompliant - log4net
nlogLogger.Debug($"The value of the parameter is: {parameter}.");       // Noncompliant - NLog
castleLogger.Debug($"The value of the parameter is: {parameter}.");     // Noncompliant - Castle.Core
serilogLogger.Debug($"The value of the parameter is: {parameter}.");    // Noncompliant - Serilog

Compliant solution


msLogger.LogDebug("The value of the parameter is: {parameter}.", parameter);    // Microsoft.Extensions.Logging
log4netLogger.DebugFormat("The value of the parameter is: {0}.", parameter);    // log4net
nlogLogger.Debug("The value of the parameter is: {parameter}.", parameter);     // NLog
castleLogger.DebugFormat("The value of the parameter is: {0}.", parameter);     // Castle.Core
serilogLogger.Debug("The value of the parameter is: {@parameter}", parameter);  // Serilog
